    Project 1.3.1: A Dangerous Situation (Cybersecurity)

  • 11/1/22-11/7/22

    • For Project 1.3.1, my partner Ali and I had to find apparent as well as hidden issues that were located in a virtual lab. From this activity, I learned how to problem solve and effectively deal with issues that could occur when protecting a system against cyber attacks. Each issue was documented here under Unit 1.3.1. The first issue we found was that there was a suspicious owner for this file and removed its permissions so that it couldn’t run any suspicious applications. Next we found a suspicious python application running in the background that could be potential malware, so we ended the task. We then detected some suspicious downloaded files in our download folder and removed them. We continued to find some suspicious files on our disk with names and owners that we didn’t recognize and removed them from the folder. Next, I found some files in the recycle bin that were suspicious and needed to be fully deleted. I restored my chrome web browser because that is a trusted file that we need to use before removing the rest of them. Then, I found that my firewall was off! This allows malware and suspicious files to be sent directly to my computer without a way to stop it. I had to turn it back on. I went on to find that my web and email protection was disabled so I re-enabled it using AVG business security so that I could be safe while browsing the web. Next, I deleted the suspicious file foo1.exe because it was an unknown python file that could have been running a malicious program. Next, I removed another suspicious python file called foo2, which could have also been running malware in the background. Lastly, I found a third foo file which was an unknown python file stored in a suspicious folder which I deleted to completely fix my computer protection and make it safe from outside attacks.
